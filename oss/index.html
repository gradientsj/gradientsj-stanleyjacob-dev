LLM systems and retrieval

Retrieval, generation, evaluation, and fine-tuning: the pieces of software built on language models.

  • llm-eval-harness

    An LLM-as-judge evaluation harness with human calibration and two-tier CI quality gating, demonstrated on grounded QA. The judge is gated on its agreement with humans (Cohen's kappa, quadratic-weighted kappa) rather than assumed correct, a deterministic lexical judge sets the floor an LLM judge has to beat, and a regression gate compares every candidate release against a frozen baseline in CI. Usage article.

    Python
  • rag-platform

    A retrieval-augmented generation service, from document ingestion and indexing through hybrid retrieval to grounded, cited answers. The design thinking behind it is worked through in the RAG system design write-up.

    Python
  • agentic-rag-mcp

    Retrieval exposed as Model Context Protocol tools, so an agent can decide when to search, refine its queries, and read what it retrieved, rather than being handed one fixed context block.

    Python
  • rust-coder-ft

    A full BF16 fine-tune of Qwen3-8B for idiomatic Rust on four H100s with FSDP2, quantized to FP8 and served with vLLM, then improved further by reinforcement learning with the compiler as the reward. Full write-up: an 8B Rust coder, fully fine-tuned.

    Python

Speech and video

Models and pipelines that listen and watch: recognition, diarization, and video understanding.

  • Mega-ASR

    A foundation speech-recognition effort built for real-world audio: seven atomic acoustic conditions composed into 54 compound scenarios and 2.6 million samples, so the training distribution covers the noise, reverb, and overlap that field audio actually contains.

    Python
  • speech-diarization-lab

    Speaker-attributed transcription: faster-whisper ASR joined to a diarization pipeline assembled from open parts, scored end to end with WER and DER implemented from scratch, and streaming speaker-labeled words a few seconds behind live. Full write-up: speaker-attributed transcription.

    Python
  • video-analysis-studio

    A video understanding studio in the SAM3 mold: text-prompted segmentation and tracking, open-vocabulary detection, and action recognition over uploaded footage.

    Python

GPU kernels

Hand-written kernels, benchmarked against the libraries they compete with. Pairs with the ML section, where key submodules get Triton and CUDA implementations beside their reference versions.

  • flash-attention-cuda

    The FlashAttention forward pass written from scratch in CUDA for the RTX 4090 (Ada, sm_89), optimized step by step from a naive baseline, with each step's speedup measured. The online-softmax trick at its core is explained on the softmax page.

    CUDA

Robotics

Learning policies for physical systems.

  • robot-imitation-lab

    Imitation-learning robot policies on open data: train and evaluate Diffusion Policy on PushT against the published reference on matched seeds, and build the success-versus-compute scaling curve. Full write-up: robot imitation learning.

    Python

Applications

Software a person can open and use, with the measurement to back it.

  • uptime-slo-monitor

    A live public status page backed by a service that probes endpoints on a schedule, computes availability and latency SLIs over rolling windows, and fires multi-window burn-rate alerts against per-service error budgets.

    TypeScript
  • chess-lab

    A desktop chess game in C++ and SFML whose move generator is verified against more than ten million published perft positions, with an alpha-beta AI, plus an AlphaZero-style network that learns the game from self-play. Full write-up: a chess lab in two halves.

    C++
  • credit-risk-model

    An 800,000-application credit-risk PD model that beats the incumbent scorecard by 0.10 Gini, with SHAP adverse-action reason codes and a fair-lending audit tested against deliberately planted bias. Full write-up: credit risk with a provable audit.

    Python
