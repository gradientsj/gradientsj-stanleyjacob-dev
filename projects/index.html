Uptime & SLO Monitor

A self hosted service that probes live endpoints on a schedule, records latency and availability in Postgres, and serves this public status page. It computes availability and latency SLIs over rolling windows, tracks error budget burn against per service objectives, and fires multi window multi burn rate alerts so a fast burn pages immediately while a slow burn only warns.

What it shows

The engineering behind it.

It's a small project, but it touches the things I care about in real systems: clear service objectives, honest measurement, and alerting that wakes you only when it should. Availability and latency SLIs are computed over rolling windows, error budgets are tracked per service against targets set in YAML, and the alerting uses a multi-window, multi-burn-rate strategy so a sudden outage pages fast while a slow leak just files a warning.

The whole thing is built like production reliability tooling. Probing runs on a schedule, every result lands in Postgres for real-time and historical uptime, and the status page degrades gracefully when the database is gone instead of crashing. A runbook covers the common failure modes and recovery steps.

It ships as Docker images, exposes Prometheus metrics for Grafana, and comes with Terraform and a Helm chart for a Kubernetes (GKE) deploy, plus a k6 load test that holds the probe path to its latency budget under load.
