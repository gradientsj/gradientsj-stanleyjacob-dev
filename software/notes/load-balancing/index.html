Load balancing and horizontal scaling

Software & architecture · architecture · Jun 2026

Vertical scaling (a bigger machine) hits a ceiling fast; horizontal scaling (more machines) is how systems grow. A load balancer makes many servers look like one, spreading requests and routing around failures.

It also does health checks and often TLS termination, but the cleanest design keeps servers stateless so any request can go to any server.

When it matters

  • Any service that outgrows a single instance.
  • High availability by removing single points of failure.
  • Rolling deploys and zero-downtime maintenance.

Algorithms

  1. Round robin: rotate through servers in order; even when servers are equal.
  2. Weighted round robin: send more traffic to bigger servers.
  3. Least connections: send to the server with the fewest active requests; good for uneven request costs.
  4. Hash or sticky: route a client consistently to the same server (e.g. for in-memory session).

Implementation

from itertools import cycle

class RoundRobin:
    def __init__(self, servers):
        self.it = cycle(servers)

    def next(self):
        return next(self.it)

# weighted: expand the list, e.g. ["A","A","B"] sends twice as much to A
# least-connections: pick min(servers, key=active_count) instead

Complexity (time and space)

Round robin is O(1) per decision with O(1) state. Least-connections is O(1) with a counter per server (or a heap for the minimum). The real constraint is keeping servers stateless so the balancer is free to pick any of them.

Worked example

rr = RoundRobin(["A", "B", "C"])
print([rr.next() for _ in range(4)])   # ['A', 'B', 'C', 'A']

Follow-up questions

  • Why prefer stateless servers? So any server can handle any request; session state lives in a shared store or a signed token, enabling free scaling and failover.
  • L4 vs L7 load balancing? L4 routes by IP and port (fast, protocol-agnostic); L7 routes by HTTP content like path or headers, enabling smarter routing at higher cost.
  • How does the balancer detect a dead server? Periodic health checks; failing servers are pulled from the pool until they recover.
  • Least connections vs round robin? Least connections handles uneven request costs better; round robin is ideal when requests are uniform and servers identical.
