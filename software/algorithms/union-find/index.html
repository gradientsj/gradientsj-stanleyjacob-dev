Union-Find (disjoint sets)

Algorithms · graphs · Jun 2026

Union-Find (a disjoint-set union) maintains a collection of disjoint sets and answers two questions fast: which set is this element in, and merge these two sets. Each set is a tree pointing at a representative root.

Two optimizations make it almost constant time: path compression flattens trees during lookups, and union by rank attaches the smaller tree under the larger.

When to use it

  • Connectivity queries as edges are added (dynamic connectivity).
  • Kruskal's minimum spanning tree.
  • Grouping or detecting cycles in an undirected graph.

Step by step

  1. Give each element a parent pointer to itself.
  2. find: follow parents to the root, compressing the path on the way up.
  3. union: find both roots; if different, attach the lower-rank root under the higher.
  4. Two elements are connected exactly when they share a root.
class UnionFind:
    def __init__(self, n):
        self.parent = list(range(n))
        self.rank = [0] * n

    def find(self, x):
        while self.parent[x] != x:
            self.parent[x] = self.parent[self.parent[x]]   # path compression
            x = self.parent[x]
        return x

    def union(self, a, b):
        ra, rb = self.find(a), self.find(b)
        if ra == rb:
            return False
        if self.rank[ra] < self.rank[rb]:
            ra, rb = rb, ra
        self.parent[rb] = ra
        if self.rank[ra] == self.rank[rb]:
            self.rank[ra] += 1
        return True

Complexity

Almost O(1) per operation: the amortized cost is the inverse Ackermann function, effectively constant. Space is O(n).
