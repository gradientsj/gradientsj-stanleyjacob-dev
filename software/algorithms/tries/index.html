Tries (prefix trees)

Algorithms · trees · Jun 2026

A trie stores strings by their characters: each edge is a letter and each path from the root spells a prefix. Looking up a word or a prefix costs time proportional to its length, independent of how many words are stored.

That makes tries the natural structure for autocomplete, spell-check, dictionaries, and word-search problems.

When to use it

  • Prefix queries: does any stored word start with this?
  • Autocomplete and typeahead.
  • Word games and dictionary lookups where many keys share prefixes.

Step by step

  1. Each node holds a map from a character to a child node, plus an end-of-word flag.
  2. Insert: walk or create a child per character, then mark the last node as a word end.
  3. Search: walk the children; the word exists only if you end on a word-end node.
  4. Prefix: the same walk, but you do not require the end flag.
class Trie:
    def __init__(self):
        self.children = {}
        self.end = False

    def insert(self, word):
        node = self
        for ch in word:
            node = node.children.setdefault(ch, Trie())
        node.end = True

    def search(self, word):
        node = self
        for ch in word:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return node.end

    def starts_with(self, prefix):
        node = self
        for ch in prefix:
            if ch not in node.children:
                return False
            node = node.children[ch]
        return True

Complexity

Insert, search, and prefix checks are O(L) for a word of length L, independent of the number of stored words. Space can be large since every character is a node; compressed tries (radix trees) reduce it.
