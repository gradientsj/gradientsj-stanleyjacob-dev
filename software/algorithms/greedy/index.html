Greedy algorithms

Algorithms · paradigm · Jun 2026

A greedy algorithm builds a solution one step at a time, always taking the choice that looks best right now and never reconsidering. When it works it is simple and fast; the catch is that it does not always work.

The classic example is interval scheduling: to fit the most non-overlapping meetings, repeatedly take the one that finishes earliest.

When to use it

  • The problem has the greedy-choice property: a local optimum leads to a global optimum.
  • Interval scheduling, Huffman coding, minimum spanning trees, fractional knapsack.
  • As a fast heuristic when an exact method is too slow (accepting approximate answers).

Step by step

  1. Find the right ordering or selection criterion (often a sort).
  2. Take the best available choice that keeps the solution valid.
  3. Never undo a choice.
  4. Justify it with an exchange argument: show swapping toward the greedy choice never hurts.
def max_non_overlapping(intervals):
    intervals.sort(key=lambda x: x[1])        # earliest finish time first
    count, end = 0, float('-inf')
    for s, e in intervals:
        if s >= end:                          # this interval starts after the last kept one
            count += 1
            end = e
    return count

Complexity

Usually dominated by the sort, O(n log n), then a linear pass. The real work is proving the greedy choice is optimal; for 0/1 knapsack, for example, greedy is wrong and you need dynamic programming.
