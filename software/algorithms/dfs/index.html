Depth-first search (DFS)

Algorithms · graphs · Jun 2024

Depth-first search explores as far as it can along each branch before backing up. It is the workhorse behind connectivity, path finding, topological order, and cycle detection.

DFS is naturally recursive (the call stack is the stack), but an explicit stack version avoids deep recursion limits on large graphs.

When to use it

  • Visiting every node or edge of a graph or tree.
  • Detecting connected components or cycles.
  • Producing a topological order, or any problem that needs to fully explore one path before another.

Step by step

  1. Mark the current node visited and record it.
  2. Recurse into each unvisited neighbor.
  3. Backtrack when a node has no unvisited neighbors.
  4. For the iterative version, push neighbors on a stack and pop to visit.
def dfs(graph, start):
    visited, order = set(), []
    def go(u):
        visited.add(u); order.append(u)
        for v in graph[u]:
            if v not in visited:
                go(v)
    go(start)
    return order

def dfs_iter(graph, start):
    visited, order, stack = set(), [], [start]
    while stack:
        u = stack.pop()
        if u in visited:
            continue
        visited.add(u); order.append(u)
        for v in reversed(graph[u]):
            if v not in visited:
                stack.append(v)
    return order

Complexity

O(V + E) time for V vertices and E edges. Space is O(V) for the visited set plus the recursion or explicit stack.
