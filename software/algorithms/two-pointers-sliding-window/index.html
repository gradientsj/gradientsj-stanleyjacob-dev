Two pointers and sliding window

Algorithms · foundations · Jun 2026

Two pointers and the sliding window turn many array and string problems from O(n squared) into O(n) by moving one or two indices across the data instead of re-scanning.

Two pointers usually walk inward from both ends (often on sorted data); a sliding window grows and shrinks a contiguous range while maintaining a running summary of what is inside it.

When to use it

  • Two pointers: pair or triplet sums in a sorted array, partitioning, reversing, merging.
  • Sliding window: longest or shortest subarray or substring meeting a condition.
  • Any time a brute force re-scans overlapping ranges you can probably slide instead.

Step by step

  1. Two pointers: start one index at each end and move them based on the comparison.
  2. Sliding window: expand the right edge to include new elements.
  3. Update the running summary (a count, sum, or last-seen map).
  4. Shrink from the left while the window violates the condition; record the best.
def two_sum_sorted(a, target):
    i, j = 0, len(a) - 1
    while i < j:
        s = a[i] + a[j]
        if s == target:
            return (i, j)
        if s < target:
            i += 1
        else:
            j -= 1
    return None

def longest_unique_substring(s):
    seen, start, best = {}, 0, 0
    for i, ch in enumerate(s):
        if ch in seen and seen[ch] >= start:
            start = seen[ch] + 1     # jump past the earlier copy
        seen[ch] = i
        best = max(best, i - start + 1)
    return best

Complexity

Both are O(n) time. Two pointers is O(1) space; the sliding window is O(k) for whatever it tracks about the window (here the set of characters).
