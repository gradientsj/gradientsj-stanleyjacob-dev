Dynamic programming

Algorithms · paradigm · Nov 2024

Dynamic programming applies when a problem has overlapping subproblems and optimal substructure: the answer is built from answers to smaller versions of the same problem, and those smaller answers repeat. You compute each once and reuse it.

The hard part is never the code; it is defining the state and the recurrence. Once those are right, the implementation is a table you fill in, top-down with memoization or bottom-up with tabulation.

When to use it

  • Counting the number of ways, or optimizing a value, over choices.
  • Problems on sequences, grids, intervals, or subsets with reusable subresults.
  • When a greedy choice is not safe but the problem still decomposes.

Step by step

  1. Define the state: what the subproblem index means.
  2. Write the recurrence: how a state depends on smaller states.
  3. Set the base cases.
  4. Choose an evaluation order so dependencies are ready.
  5. Read the answer out of the final state.
import bisect

# 0/1 knapsack: each item used at most once
def knapsack(weights, values, cap):
    dp = [0] * (cap + 1)
    for w, v in zip(weights, values):
        for c in range(cap, w - 1, -1):       # high to low so each item is used once
            dp[c] = max(dp[c], dp[c - w] + v)
    return dp[cap]

# longest increasing subsequence in O(n log n)
def lis(nums):
    tails = []
    for x in nums:
        i = bisect.bisect_left(tails, x)
        if i == len(tails):
            tails.append(x)
        else:
            tails[i] = x
    return len(tails)

# edit (Levenshtein) distance between two strings
def edit_distance(a, b):
    m, n = len(a), len(b)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(m + 1): dp[i][0] = i
    for j in range(n + 1): dp[0][j] = j
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if a[i - 1] == b[j - 1]:
                dp[i][j] = dp[i - 1][j - 1]
            else:
                dp[i][j] = 1 + min(dp[i - 1][j], dp[i][j - 1], dp[i - 1][j - 1])
    return dp[m][n]

# fewest coins to make an amount
def coin_change(coins, amount):
    INF = float('inf')
    dp = [0] + [INF] * amount
    for c in range(1, amount + 1):
        for coin in coins:
            if coin <= c:
                dp[c] = min(dp[c], dp[c - coin] + 1)
    return dp[amount] if dp[amount] != INF else -1

Complexity

Cost is the number of states times the work per state. Knapsack is O(n times cap); edit distance is O(m times n); coin change is O(amount times coins); LIS here is O(n log n). Memory often reduces to one or two rows.
