Sorting: merge sort and quicksort

Algorithms · foundations · Jun 2026

Merge sort and quicksort are the two comparison sorts worth knowing cold. Both are divide and conquer, but they split the work differently and have different trade-offs.

Merge sort splits the array in half, sorts each half, and merges them. Quicksort partitions around a pivot so everything smaller is on the left, then recurses on each side.

When to use it

  • Merge sort: when you need a stable sort or are sorting linked lists or external data.
  • Quicksort: when you want in-place sorting with great average-case speed and cache behavior.
  • In practice, library sorts (Timsort in Python) combine merge sort with insertion sort.

Step by step

  1. Merge sort: split in half, recursively sort each half, merge the two sorted halves.
  2. Merge: walk both halves with two pointers, always taking the smaller front element.
  3. Quicksort: choose a pivot and partition so smaller elements come before it.
  4. Recurse on the left and right partitions around the pivot's final position.
def merge(left, right):
    out, i, j = [], 0, 0
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            out.append(left[i]); i += 1
        else:
            out.append(right[j]); j += 1
    out.extend(left[i:]); out.extend(right[j:])
    return out

def merge_sort(a):
    if len(a) <= 1:
        return a
    mid = len(a) // 2
    return merge(merge_sort(a[:mid]), merge_sort(a[mid:]))

def partition(a, lo, hi):
    pivot, i = a[hi], lo
    for j in range(lo, hi):
        if a[j] < pivot:
            a[i], a[j] = a[j], a[i]; i += 1
    a[i], a[hi] = a[hi], a[i]
    return i

def quicksort(a, lo=0, hi=None):
    if hi is None:
        hi = len(a) - 1
    if lo >= hi:
        return a
    p = partition(a, lo, hi)
    quicksort(a, lo, p - 1)
    quicksort(a, p + 1, hi)
    return a

Complexity

Merge sort: O(n log n) time always, O(n) extra space, stable. Quicksort: O(n log n) average, O(n squared) worst case (mitigated by random or median pivots), O(log n) stack space, in place but not stable.
