Heaps and priority queues

Algorithms · trees · Jun 2026

A heap is a complete binary tree (stored in an array) where every parent is smaller than its children (a min-heap). That invariant gives O(1) access to the minimum and O(log n) insertion and removal.

It is the right structure whenever you repeatedly need the current best element: scheduling, Dijkstra and Prim, streaming top-K, and merging sorted streams.

When to use it

  • Repeatedly extracting the smallest or largest element.
  • Top-K or streaming-median problems.
  • Driving Dijkstra, Prim, or any best-first search.

Step by step

  1. Keep the heap as an array; children of index i are at 2i+1 and 2i+2.
  2. Push: append, then sift up while smaller than the parent.
  3. Pop: swap root with the last element, remove it, then sift down.
  4. For top-K largest, keep a min-heap of size k and evict the smallest.
import heapq

def top_k(nums, k):
    # keep a min-heap of size k: its root is the kth largest seen
    h = []
    for x in nums:
        heapq.heappush(h, x)
        if len(h) > k:
            heapq.heappop(h)          # drop the smallest, keep the top k
    return sorted(h, reverse=True)

# heapq is a min-heap; push negatives for a max-heap, or use heapq.nlargest

Complexity

Push and pop are O(log n); peeking the min is O(1). Top-K of n items with a size-k heap is O(n log k), better than sorting everything when k is small.
