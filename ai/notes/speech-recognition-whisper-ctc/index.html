Speech recognition: Whisper and CTC

Notes · automatic speech recognition · Jun 2026

Automatic speech recognition turns a waveform into text. Two families dominate: connectionist temporal classification (CTC) and encoder-decoder models like Whisper. They solve the same alignment problem in very different ways. This is the backbone of my speech transcription and diarization project.

From waveform to features

Raw audio is first turned into a log-mel spectrogram, a time-by-frequency image that mirrors how the ear weights pitch. Everything downstream operates on this representation rather than the raw samples.

CTC: alignment without labels

The hard part of ASR is that you have audio frames and a transcript, but no per-frame labels. CTC solves this by adding a blank symbol and summing the probability over every possible alignment of text to frames. It assumes outputs are conditionally independent given the audio, which makes it fast and naturally streaming, at some cost to accuracy on context-dependent spellings.

Whisper: encoder-decoder with attention

Whisper takes the sequence-to-sequence route: a transformer encoder reads the spectrogram, and a decoder generates text tokens autoregressively, attending back to the audio. Trained on a very large amount of weakly-supervised multilingual audio, it is robust to noise and accents and produces punctuation, casing, and even translation for free. The cost is that decoding is sequential and heavier than CTC.

Making it fast

In practice I run Whisper through faster-whisper (CTranslate2), which quantizes and fuses the model for a large speedup, plus voice-activity detection to skip silence and word-level timestamps for alignment. That is what makes near real-time transcription on a single GPU practical.

Step by step: CTC greedy decode

  1. Take the most likely symbol at each time frame (argmax).
  2. Collapse runs of the same symbol into one.
  3. Remove the blank symbol.
  4. What remains is the decoded sequence.
  5. Beam search keeps several hypotheses instead of one, trading speed for accuracy.
import numpy as np

def ctc_greedy_decode(logits, blank=0):
    """logits: (T, vocab). Greedy best-path decode."""
    best = logits.argmax(axis=-1)        # most likely symbol per frame
    out, prev = [], None
    for s in best:
        if s != prev and s != blank:     # collapse repeats, then drop blanks
            out.append(int(s))
        prev = s
    return out

# frames whose argmax is [1, 1, blank, 2] decode to [1, 2]
