How diffusion models generate images

Notes · diffusion generative models · Jun 2026

Diffusion models learn to turn noise into data by reversing a process that turns data into noise. It sounds circular, but it gives a stable training target and state-of-the-art image quality.

The forward process

Take a real image and add a little Gaussian noise, repeatedly, over many steps. After enough steps the image is indistinguishable from pure noise. This direction needs no learning, it is just a fixed schedule of adding noise, and crucially you can jump to any noise level in one step.

The reverse process

The model learns to undo one step of noising. In practice it is trained to predict the noise that was added to a sample, and the loss is a plain mean-squared error between the predicted and actual noise: 

loss = || ε − ε_θ(x_t, t) ||²

That simple regression objective is the whole training signal. A U-Net (or, increasingly, a transformer) does the prediction, conditioned on the timestep so it knows how noisy the input is.

Sampling

To generate, start from random noise and apply the learned denoiser step by step. DDPM does this with many small stochastic steps; DDIM makes it deterministic and lets you trade quality for far fewer steps.

Text conditioning and latent diffusion

To steer generation with a prompt, the text is encoded and fed to the denoiser through cross-attention. Classifier-free guidance then amplifies the prompt's influence by extrapolating between the conditional and unconditional predictions. Stable Diffusion adds one more trick: latent diffusion, running the whole process in the compressed latent space of a VAE instead of raw pixels, which is what makes high-resolution generation affordable.

A hard generative problem becomes a sequence of easy denoising problems, each one a small, well-behaved regression.

Step by step

  1. Fix a noise schedule and precompute the cumulative product of alphas.
  2. Forward: jump any image to noise level t in one closed-form step.
  3. Train the model to predict the noise that was added (an MSE loss).
  4. Reverse: each step subtracts the predicted noise and adds a little back.
  5. Sample by starting from pure noise and denoising from T down to 0.
import numpy as np

T = 1000
betas = np.linspace(1e-4, 0.02, T)            # noise schedule
alphas = 1.0 - betas
alpha_bar = np.cumprod(alphas)

def q_sample(x0, t, noise):                    # forward: add t steps of noise
    return np.sqrt(alpha_bar[t]) * x0 + np.sqrt(1 - alpha_bar[t]) * noise

def training_loss(model, x0):
    t = np.random.randint(T)
    noise = np.random.randn(*x0.shape)
    xt = q_sample(x0, t, noise)
    return ((model(xt, t) - noise) ** 2).mean()    # predict the noise

def p_sample_step(model, x, t):                # reverse: one denoising step
    eps = model(x, t)
    mean = (x - betas[t] / np.sqrt(1 - alpha_bar[t]) * eps) / np.sqrt(alphas[t])
    return mean + (np.sqrt(betas[t]) * np.random.randn(*x.shape) if t > 0 else 0)

def sample(model, shape):
    x = np.random.randn(*shape)                # start from pure noise
    for t in reversed(range(T)):
        x = p_sample_step(model, x, t)
    return x
