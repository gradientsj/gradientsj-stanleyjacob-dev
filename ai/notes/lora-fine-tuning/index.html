Parameter-efficient fine-tuning with LoRA

Notes · adapting large models cheaply · Jun 2026

Fully fine-tuning a large model means updating every weight and storing optimizer state for all of them, which is expensive and produces a full-size copy per task. LoRA sidesteps almost all of that. I use it in my text-to-image project to teach a custom style.

The idea

Freeze the pretrained weights and inject a small, trainable low-rank update into the linear layers. Instead of learning a full matrix update, learn two thin matrices whose product has rank r, with r far smaller than the layer width: 

output = W·x + (B·A)·x        # W frozen; only A, B are trained
# A: r×d, B: d×r, with r ≪ d

The bet, supported by the original paper, is that the change a model needs in order to adapt to a new task is itself low-rank, so a few extra parameters capture most of it.

Why it is useful

  • Tiny trainable footprint: often well under 1% of the model's parameters.
  • Swappable adapters: one frozen base, many small task adapters you can load on demand.
  • No inference penalty: the update can be merged back into the weights once trained.

QLoRA

QLoRA pushes this further by quantizing the frozen base model to 4-bit and training the LoRA adapters on top. The base barely moves in memory, so models that would never fit otherwise can be fine-tuned on a single consumer GPU.

In practice you rarely need to move all the weights to specialize a model. A small, well-placed update usually does the job.

Step by step

  1. Wrap a frozen pretrained linear layer.
  2. Add two small matrices A and B of rank r.
  3. Initialize B to zero so training starts exactly at the base model.
  4. Forward adds the scaled low-rank update to the frozen output.
  5. Only A and B get gradients; merge B times A into W to drop the inference cost.
import torch
import torch.nn as nn

class LoRALinear(nn.Module):
    def __init__(self, in_f, out_f, r=8, alpha=16):
        super().__init__()
        self.base = nn.Linear(in_f, out_f)
        for p in self.base.parameters():
            p.requires_grad_(False)            # freeze the pretrained weights
        self.A = nn.Parameter(torch.randn(r, in_f) * 0.01)
        self.B = nn.Parameter(torch.zeros(out_f, r))   # zero: starts as a no-op
        self.scale = alpha / r

    def forward(self, x):
        return self.base(x) + self.scale * (x @ self.A.t() @ self.B.t())
