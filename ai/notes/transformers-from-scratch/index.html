Transformers from scratch

Notes · the transformer architecture · Jun 2026

The transformer replaced recurrence with a single idea: let every token look directly at every other token. That one change is what made today's language models trainable at scale, because the computation parallelizes across the whole sequence instead of stepping through it one position at a time.

Scaled dot-product attention

Each token is projected into a query, a key, and a value. A token's output is a weighted average of all values, where the weights come from how well its query matches each key: 

attention(Q, K, V) = softmax(Q Kᵀ / √d_k) V

The √d_k term matters: without it, dot products grow with dimension, push the softmax into saturated regions, and kill the gradients. Dividing by the square root of the key dimension keeps the scores in a sane range.

Multi-head attention

One attention function can only express one notion of "relevance." So we run several in parallel, each in its own lower-dimensional subspace, then concatenate. Different heads learn different relations: some track syntax, some track long-range coreference, some mostly copy the previous token.

Position has to be added back

Attention is permutation-invariant: shuffle the tokens and the output shuffles with them. Language is not, so position has to be injected. The original paper used fixed sinusoids; modern LLMs use rotary embeddings (RoPE), which rotate queries and keys by an angle proportional to position and generalize better to longer contexts.

The block

A transformer block is just attention followed by a position-wise MLP, each wrapped in a residual connection and layer normalization. Modern models put the norm before each sublayer (pre-norm), which makes very deep stacks stable to train. 

x = x + attention(norm(x))
x = x + mlp(norm(x))

For language modeling the attention is causal: a mask stops each position from attending to future tokens, so the model can be trained to predict the next token everywhere at once.

Why it scales

An RNN's hidden state is a sequential bottleneck. Attention has no such dependency: every position is computed in parallel, which is exactly what saturates a GPU. The cost is the all-pairs interaction, quadratic in sequence length, which is why long-context work is mostly about making attention cheaper.

The clearest way to internalize all of this is to implement it: a tokenizer, an embedding table, a stack of these blocks, and a training loop on next-token prediction. Once the shapes line up and the loss goes down, the architecture becomes concrete.

Step by step

  1. Project each token into query, key, and value vectors.
  2. Score every query against every key with a scaled dot product, dividing by the square root of the key dimension.
  3. Optionally apply a causal mask so a position cannot attend to the future.
  4. Softmax the scores into weights that sum to 1.
  5. Return the weighted sum of the values.
  6. For multiple heads, split into subspaces, attend in parallel, concatenate, and project out.
import numpy as np

def softmax(x, axis=-1):
    x = x - x.max(axis=axis, keepdims=True)
    e = np.exp(x)
    return e / e.sum(axis=axis, keepdims=True)

def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.swapaxes(-1, -2) / np.sqrt(d_k)   # (..., q, k)
    if mask is not None:                             # keep where mask is True
        scores = np.where(mask, scores, -1e9)
    weights = softmax(scores, axis=-1)
    return weights @ V, weights

def multi_head_attention(X, Wq, Wk, Wv, Wo, num_heads):
    seq, d_model = X.shape
    d_head = d_model // num_heads
    Q, K, V = X @ Wq, X @ Wk, X @ Wv
    split = lambda t: t.reshape(seq, num_heads, d_head).transpose(1, 0, 2)
    out, _ = scaled_dot_product_attention(split(Q), split(K), split(V))
    out = out.transpose(1, 0, 2).reshape(seq, d_model)   # concat heads
    return out @ Wo
