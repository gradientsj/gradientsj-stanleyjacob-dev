Transformers from scratch

Notes · the transformer architecture · Jun 2026

The transformer is the architecture behind essentially every modern language model, and it rests on a single idea: instead of processing a sequence one step at a time, let every position look directly at every other position and decide for itself what is relevant. Earlier sequence models such as recurrent networks and LSTMs carried information forward through a hidden state that was updated token by token, which made them inherently sequential and awkward to parallelize. The transformer throws out that recurrence and replaces it with attention, a mechanism that computes, for each token, a weighted blend of all the other tokens. Because every position is computed independently of the others, the whole sequence can be processed at once on a GPU, and that is the practical reason transformers scaled when recurrent models stalled.

It helps to keep the shapes in mind throughout. A sequence of n tokens is first turned into n vectors of some dimension d, giving an n-by-d matrix. Every layer of the transformer maps that matrix to another matrix of exactly the same shape, refining each token's vector using context drawn from the rest of the sequence. Stacking many such layers, interleaved with small feed-forward networks, is the entire model. The sophistication is concentrated almost entirely in the attention step, so that is the part worth understanding in detail.

Scaled dot-product attention

Attention begins by projecting each token vector into three separate vectors with three learned weight matrices: a query, a key, and a value. The intuition is that the query encodes what a token is looking for, the key encodes what a token has to offer, and the value is the information a token will actually contribute if it is selected. To decide how strongly token i should attend to token j, we take the dot product of i's query with j's key; a large dot product means the two are a good match. Doing this for every pair produces an n-by-n matrix of raw compatibility scores.

Those scores are divided by the square root of the key dimension and then passed through a softmax, which turns each row into a set of weights that are positive and sum to one. The scaling step is easy to overlook but important: as the dimension grows, dot products grow in magnitude, which pushes the softmax toward a near one-hot output where its gradient is almost zero. Dividing by the square root of the key dimension keeps the scores in a range where the softmax stays smooth and the model can keep learning. Each token's output is then the weighted sum of all the value vectors using its row of weights, so a token literally becomes a blend of the tokens it found most relevant.

attention(Q, K, V) = softmax(Q Kᵀ / √d_k) V

Multi-head attention

A single attention function can only express one notion of relevance at a time, which is limiting when language has many kinds of relationships at once. Multi-head attention runs several attention computations in parallel, each with its own learned projections into a lower-dimensional subspace, then concatenates their outputs and passes them through a final linear layer. The heads end up specializing in practice: some track local grammatical structure, some resolve long-range references like which noun a pronoun points to, and some mostly copy a nearby token. Splitting the work this way gives the layer a far richer view of context than one large head would, at the same total cost.

Position has to be added back

Attention on its own has no sense of order. If you shuffle the input tokens, the outputs shuffle the same way, because the mechanism only ever compares content, never positions. Since word order plainly carries meaning, position must be injected explicitly. The original transformer added fixed sinusoidal vectors to the token embeddings, but most modern large language models use rotary position embeddings (RoPE), which rotate the query and key vectors by an angle proportional to their position. A nice property of RoPE is that relative offsets fall directly out of the dot product, and it extrapolates to longer sequences than were seen in training better than absolute encodings do.

The block: residuals, normalization, and the MLP

A transformer block wraps attention and a small feed-forward network in two patterns that are what actually make deep stacks trainable. Each sublayer adds its output back onto its input through a residual connection, so gradients have a short path all the way to the bottom of the network and each layer only has to learn a refinement rather than a full transformation from scratch. Layer normalization rescales every token vector to a stable mean and variance, and modern models apply it before each sublayer (pre-norm), which keeps the signal well-behaved even at dozens or hundreds of layers.

The feed-forward network is deceptively simple: two linear layers with a nonlinearity in between, applied to each position independently. It is usually several times wider than the model dimension, and a large fraction of the model's raw parameter count and capacity lives there rather than in attention. A useful mental split is that attention moves information between positions, while the feed-forward network does per-position computation on the information that arrived.

x = x + attention(norm(x))
x = x + mlp(norm(x))

Causal masking and training

For language modeling the model must predict the next token without being allowed to see it, so attention is made causal: position i may attend to positions up to and including i, but never beyond. In practice this is a triangular mask that sets the forbidden scores to negative infinity before the softmax, so their weights become exactly zero. A pleasant consequence is training efficiency: with the mask in place, one forward pass over a length-n sequence produces a next-token prediction at every position simultaneously, turning a single example into n separate training signals and making pretraining remarkably compute-efficient.

Why it scales, and what it costs

The reason transformers took over is throughput. A recurrent network's hidden state is a hard sequential dependency, so token t cannot start until token t-1 finishes, which leaves a parallel processor mostly idle. Attention has no such dependency, so the large matrix multiplications for the entire sequence run at once and saturate the hardware. The price is that attention compares every token with every other token, so both compute and memory grow with the square of the sequence length, which is exactly why long-context research pours so much effort into sparse attention patterns, sliding windows, and caching.

Step by step

  1. Project each token into query, key, and value vectors.
  2. Score every query against every key with a scaled dot product, dividing by the square root of the key dimension.
  3. Optionally apply a causal mask so a position cannot attend to the future.
  4. Softmax the scores into weights that sum to one.
  5. Return the weighted sum of the values.
  6. For multiple heads, split into subspaces, attend in parallel, concatenate, and project out.
import numpy as np

def softmax(x, axis=-1):
    x = x - x.max(axis=axis, keepdims=True)
    e = np.exp(x)
    return e / e.sum(axis=axis, keepdims=True)

def scaled_dot_product_attention(Q, K, V, mask=None):
    d_k = Q.shape[-1]
    scores = Q @ K.swapaxes(-1, -2) / np.sqrt(d_k)   # (..., q, k)
    if mask is not None:                             # keep where mask is True
        scores = np.where(mask, scores, -1e9)
    weights = softmax(scores, axis=-1)
    return weights @ V, weights

def multi_head_attention(X, Wq, Wk, Wv, Wo, num_heads):
    seq, d_model = X.shape
    d_head = d_model // num_heads
    Q, K, V = X @ Wq, X @ Wk, X @ Wv
    split = lambda t: t.reshape(seq, num_heads, d_head).transpose(1, 0, 2)
    out, _ = scaled_dot_product_attention(split(Q), split(K), split(V))
    out = out.transpose(1, 0, 2).reshape(seq, d_model)   # concat heads
    return out @ Wo

Complexity (time and space)

For a sequence of length n and model dimension d, self-attention costs O(n squared times d) time, dominated by forming the n-by-n score matrix and multiplying it with the values, and O(n squared) memory to hold that matrix. The feed-forward sublayer costs O(n times d squared). At inference, caching the keys and values of previous tokens lets each new token be produced in about O(n times d) work instead of recomputing the whole sequence, which is what makes autoregressive generation practical.

Worked example

Attention preserves the sequence shape and produces a valid probability distribution per token. Here three tokens of dimension four pass through a two-head layer, and the raw attention weights for each token sum to one:

import numpy as np
np.random.seed(0)

X = np.random.randn(3, 4)                    # 3 tokens, model dim 4
Wq, Wk, Wv, Wo = [np.random.randn(4, 4) for _ in range(4)]

out = multi_head_attention(X, Wq, Wk, Wv, Wo, num_heads=2)
print(out.shape)                            # (3, 4)  -- shape in == shape out

_, weights = scaled_dot_product_attention(X, X, X)
print(weights.sum(axis=-1))                 # [1. 1. 1.]  -- each row is a distribution

Follow-up questions

  • Why divide by the square root of the key dimension? Without it, dot products grow with dimension and saturate the softmax, where the gradient vanishes; the scaling keeps scores in a trainable range.
  • What are the time and memory costs of attention? O(n squared times d) compute and O(n squared) memory in the sequence length n, which is why long contexts are expensive.
  • Why do transformers parallelize better than RNNs? An RNN's hidden state forces strictly sequential computation, while attention computes every position at once and saturates a GPU.
  • If attention ignores order, how does the model know it? Position is injected explicitly, through sinusoidal encodings or rotary embeddings (RoPE).
  • What changes between training and generation? Training scores all positions in parallel under a causal mask; generation emits one token at a time and caches past keys and values to avoid recomputation.
